# SHML Platform — Hermes Context

## Workspace Layout
- `inference/` — LLM, image gen, agent-service (qwopus-coding on :8099)
- `mlflow-server/` — MLflow tracking stack
- `ray_compute/` — Ray distributed compute
- `monitoring/` — Prometheus, Grafana, Loki, Traefik
- `scripts/` — Deploy, MCP servers, Hermes dispatch, platform tools
- `libs/` — Shared Python libraries
- `docs/obsidian-vault/` — Research notes, decisions, experiments

## GPU Allocation
- RTX 3090 Ti (cuda:0, 24GB): qwopus-coding / training (yields)
- RTX 2070 (cuda:1, 8GB): Qwen3-VL-8B, z-image, embeddings

## Critical Patterns
- **Service management:** Always use `./start_all_safe.sh` or `task` commands. Never raw docker-compose.
- **Traefik priority:** Application routers MUST use priority `2147483647`.
- **Secrets:** Never hardcode. Use `os.environ["KEY"]` or Docker secrets.
- **FastAPI:** Always async handlers, type annotations, Pydantic models.
- **uv:** Use `uv pip install --system` in Dockerfiles, never pip.

## MCP Integrations
- `gitlab` — Issue management (project ID=2 for platform, ID=3 for robotics)
- `prometheus` — Cluster metrics via PromQL
- `obsidian-vault` — Research notes filesystem
- `shml-platform` — Training status, GPU info, MLflow queries
- `gitnexus` — Code intelligence knowledge graph

## Key Endpoints
- http://localhost:8080 — MLflow UI
- http://localhost:3002 — Ray UI
- http://localhost:8099/v1 — Agent-service (Hermes LLM backend)
- /api/llm/v1/chat/completions — Qwen3-VL
- /api/image/v1/generate — Z-Image

## Session Tracking
- CLI sessions: `~/.hermes/sessions/` + `~/.hermes/state.db`
- Gateway sessions: Same SQLite store, keyed by platform+chat_id
- Workspace UI: Reads from gateway API on :8642
