# ML Platform Unified Setup Script

## Overview

The `setup.sh` script is a comprehensive, zero-to-production automation that handles everything needed to deploy the ML Platform on a fresh system or reconfigure an existing installation.

## What It Does

### 🔍 Phase 1: System Dependencies
- Checks for Docker Engine and Docker Compose
- Verifies NVIDIA drivers and GPU detection
- Validates NVIDIA Container Toolkit installation
- Checks for Tailscale (optional) and PostgreSQL client
- **Interactive**: Offers to install missing dependencies with detailed prompts
- **Intelligent**: Provides manual installation guidance for drivers

### 🌐 Phase 2: Network Configuration
- Detects local and Tailscale IP addresses
- Creates Docker bridge network (`ml-platform`) if missing
- **Interactive**: Offers to set up Tailscale if not configured
- **Fallback**: Uses local IP if Tailscale unavailable

### 🔐 Phase 3: Password Configuration
- **User-Facing Services** (you'll access these):
  - Grafana admin password (for monitoring dashboards)
  - FusionAuth admin email/password (for OAuth/SSO)
- **Database Passwords** (auto-generated by default):
  - MLflow database
  - Ray database
  - FusionAuth database
  - Shared database
- **Options**: Choose between auto-generation or custom passwords
- **Context**: Each prompt explains what the service does and where it's used

### 📝 Phase 4: Environment Files
- Backs up existing `.env` files with timestamps
- Creates `.env` files from templates
- Populates all environment variables across:
  - Main `.env` (FusionAuth, Grafana, shared services)
  - `ray_compute/.env` (Ray services, APIs)
  - `mlflow-server/.env` (MLflow configuration)
- Uses Perl for reliable updates (handles special characters)

### 🔒 Phase 5: Secret Files
- Creates Docker secret files in `secrets/` directories
- Sets secure permissions (600)
- Generates comprehensive `CREDENTIALS.txt` with:
  - All access URLs
  - All usernames and passwords
  - Database connection strings
  - Security reminders
- **Zero-knowledge validation**: Verifies secrets without revealing content

### ✅ Phase 6: Pre-Flight Validation
- Validates all `docker-compose.yml` files
- Zero-knowledge checks on all secret files
- Tests GPU accessibility in Docker
- Checks port availability (80, 443, 5432, 5433, 6379, 9000, 8090)
- **Interactive**: Offers to stop conflicting services
- **Flexible**: Continues on warnings (won't block on non-critical issues)

### 🚀 Phase 7: Service Startup
Orchestrated startup in dependency order:
1. **Infrastructure** (30s wait):
   - Traefik (API gateway)
   - Redis cache
   - PostgreSQL databases
   - Node Exporter, cAdvisor

2. **Core Applications** (60s wait):
   - MLflow server
   - Ray head node
   - FusionAuth server

3. **APIs & Workers** (30s wait):
   - MLflow API and Nginx
   - Ray Compute API
   - FusionAuth background jobs

4. **Monitoring** (20s wait):
   - Prometheus (MLflow + Ray)
   - Grafana (MLflow + Ray)

### 🏥 Phase 8: Health Monitoring
- Displays container status
- Tests HTTP endpoints
- Notes on startup timing
- Provides monitoring commands

### 📊 Phase 9: Summary
- Statistics (errors, warnings, fixes)
- Access points with URLs
- Credential file location
- Useful commands
- Security reminders

## Quick Start

### Fresh Installation
```bash
cd /home/axelofwar/Projects/shml-platform/shml-platform
sudo ./setup.sh
```

### What To Expect

1. **Dependency Check** (~2 minutes if installation needed)
   - Will prompt to install Docker, NVIDIA toolkit, etc.
   - Follow prompts for missing dependencies

2. **Network Setup** (~30 seconds)
   - Detects IPs
   - Optional Tailscale setup

3. **Password Configuration** (~2-5 minutes)
   - You'll be prompted for Grafana password
   - You'll be prompted for FusionAuth admin email/password
   - Choose auto-generate or custom for database passwords

4. **Environment Setup** (~30 seconds)
   - Automatic backup and configuration

5. **Pre-Flight Checks** (~1 minute)
   - Validation with interactive fixes

6. **Service Startup** (~3 minutes)
   - Wait for orchestrated startup
   - Monitor progress

7. **Done!**
   - Platform ready to use
   - Check `CREDENTIALS.txt` for access details

## Common Workflows

### Re-run Setup (Reconfigure)
```bash
sudo ./setup.sh
```
- Backs up existing config automatically
- Can change passwords
- Restarts services

### Stop All Services
```bash
sudo ./stop_all.sh
```
- Creates backup before stopping
- Cleans up volumes
- Prunes build cache

### Check Service Status
```bash
sudo docker ps
```

### View Service Logs
```bash
sudo docker logs ml-platform-traefik
sudo docker logs mlflow-server
sudo docker logs ray-head
```

### Restart Single Service
```bash
sudo docker compose restart mlflow-server
```

## Troubleshooting

### Script Fails on Dependency Check
- Install Docker manually: https://docs.docker.com/engine/install/
- Install NVIDIA drivers: https://www.nvidia.com/Download/index.aspx
- Reboot after driver installation

### GPUs Not Detected
```bash
nvidia-smi  # Should show GPUs
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

### Port Already in Use
- Script will prompt to stop conflicting services
- Or manually: `sudo docker compose down`

### Services Not Starting
- Wait 2-3 minutes (they're slow to start)
- Check logs: `sudo docker logs <container-name>`
- Review TROUBLESHOOTING.md for common issues

### Network Issues
```bash
# Recreate network
sudo docker network rm ml-platform
sudo docker network create ml-platform --driver bridge --subnet 172.30.0.0/16
```

### Traefik Routing Issues
See LESSONS_LEARNED.md - API routes need priority 2147483647

### Ray Memory Allocation
See LESSONS_LEARNED.md - Object store must be < container memory

## Architecture Decisions

### Why Orchestrated Startup?
Services have dependencies:
- Databases must be ready before apps
- Apps must be ready before APIs
- All must be ready before monitoring

### Why Zero-Knowledge Validation?
- Checks secrets exist and have correct length
- Never logs or displays actual secrets
- Maintains security while validating

### Why Perl for Env Updates?
- Handles special characters in passwords
- More reliable than sed for complex replacements
- Cross-platform compatible

### Why Separate Password Prompts?
- User-facing vs. system-facing distinction
- Clear context for what each password does
- Option to auto-generate what you don't need to remember

## Known Issues & Mitigations

### Issue: MLflow Secret File Naming
**Symptom**: MLflow looks for `mlflow_db_password.txt` vs `db_password.txt`

**Mitigation**: Script creates both `mlflow-server/secrets/db_password.txt` (correct)

### Issue: Ray /tmp/ray Permissions
**Symptom**: Ray head can't create session directory

**Mitigation**: Monitoring phase will show warnings; check Ray container user permissions

### Issue: Service Startup Timing
**Symptom**: Services report "not responding" immediately

**Mitigation**: Script notes 2-3 minute startup time; health checks are informational only

## Advanced Configuration

### Custom Subnet
Edit `docker-compose.yml`:
```yaml
networks:
  ml-platform:
    external: true
    # Change to: ipam: config: subnet: 172.31.0.0/16
```

### Different Ports
Edit `.env` before running:
```bash
TRAEFIK_HTTP_PORT=8080
TRAEFIK_HTTPS_PORT=8443
```

### Skip Tailscale
Just say "no" when prompted, or press Ctrl+C during Tailscale setup.

## Security Best Practices

1. **Credentials File**: Back up to encrypted storage
2. **FusionAuth**: Review security settings after first login
3. **Git**: Add to `.gitignore`: `.env`, `CREDENTIALS.txt`, `secrets/`
4. **Backups**: Script creates backups in `backups/env_backups/`
5. **Permissions**: Script sets 600 on all secret files
